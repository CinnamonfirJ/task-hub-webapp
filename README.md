# Task Hub Web App

Welcome to the Task Hub Web App repository! Task Hub is a comprehensive platform designed to connect task posters with skilled taskers. The platform features task management, bidding, real-time chat, identity verification, and a robust wallet system supporting both fiat and Stellar cryptocurrency payments with escrow capabilities.

## 🛠 Tech Stack

- **Framework**: [Next.js](https://nextjs.org/) (App Router)
- **Language**: TypeScript
- **Styling**: [Tailwind CSS](https://tailwindcss.com/) v4 + [Shadcn UI](https://ui.shadcn.com/)
- **Animations**: Framer Motion
- **State Management & Data Fetching**: [React Query](https://tanstack.com/query/latest) (`@tanstack/react-query`)
- **Authentication**: Custom Auth + Google OAuth (`@react-oauth/google`)
- **Form Management**: React Hook Form + Zod for validation
- **Icons**: Lucide React
- **Identity Verification**: Didit

## 📂 Project Architecture

The codebase follows a clean, modular architecture separating the UI from the data fetching layer:

- **`app/`**: Next.js App Router definitions. Uses Route Groups (e.g., `(main)`, `(auth)`, `(onboarding)`) to organize layouts and routing logic cleanly.
- **`components/`**: Reusable React components.
  - **`components/ui/`**: Base primitive components generated by Shadcn UI.
  - **Feature-specific components**: (e.g., `WithdrawFundsModal.tsx`, `CloudinaryUpload.tsx`).
- **`hooks/`**: Custom React Query hooks. **This is the core of our state management.** Every data entity (tasks, wallet, user, chat) has a corresponding hook file (e.g., `useWallet.ts`, `useTasks.ts`) that manages caching, invalidation, and background updates.
- **`lib/api/`**: The Service Layer. Contains pure functions that make `fetch` requests to our backend (e.g., `walletApi` in `wallet.ts`). These functions are consumed exclusively by the `hooks/`.
- **`types/`**: Global TypeScript interfaces for API payloads and database models.
- **`public/`**: Static assets like images and custom fonts.

## 🚀 Getting Started

### Prerequisites
- Node.js (v18.17.0 or higher)
- npm or yarn

### Installation
1. **Clone the repository:**
   ```bash
   git clone <repository-url>
   cd task-hub-webapp
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Environment Setup:**
   Create a `.env.development` (or `.env`) file in the root directory and populate it with the necessary variables (ask a team member for the exact values):
   ```env
   NEXT_PUBLIC_API_BASE_URL=http://localhost:3009 # Or the staging/prod backend URL
   NEXT_PUBLIC_ENV=development
   NEXT_PUBLIC_APP_URL=http://localhost:3000
   
   # Didit Integration
   NEXT_PUBLIC_DIDIT_API_KEY=your_didit_api_key
   NEXT_PUBLIC_DIDIT_WEBHOOK_SECRET=your_webhook_secret
   NEXT_PUBLIC_DIDIT_WORKFLOW_ID=your_workflow_id
   NEXT_PUBLIC_DIDIT_VERIFICATION_URL=your_verification_url
   
   # Auth
   NEXT_PUBLIC_GOOGLE_CLIENT_ID=your_google_client_id
   ```

### Running the Application
Start the development server:
```bash
npm run dev
```
Open [http://localhost:3000](http://localhost:3000) in your browser.

## 🛠 How to Add New Features

When building a new feature (e.g., "Reviews"), follow this standard data flow pattern:

1. **Define Types (`types/`)**
   Create the TypeScript interfaces for the expected API response and request payloads.
   *Example: `types/review.ts`*

2. **Create API Service (`lib/api/`)**
   Create pure async functions that communicate with the backend endpoints.
   *Example: Add `lib/api/reviews.ts` with `getReviews` and `postReview`.*

3. **Create Custom Hooks (`hooks/`)**
   Wrap your API service calls in React Query hooks (`useQuery` for fetching, `useMutation` for modifying). Handle cache invalidation here.
   *Example: Add `hooks/useReviews.ts` exporting `useReviews` and `useSubmitReview`.*

4. **Build UI Components (`components/`)**
   Create the visual components. Use Shadcn components from `components/ui/` and Tailwind for styling. Consume the data using the hooks you created in step 3.
   *Example: `components/ReviewList.tsx`.*

5. **Add the Route (`app/`)**
   Create the `page.tsx` file in the appropriate App Router directory to render your new component.

## 📏 Best Practices & Conventions

- **Server vs Client Components**: Next.js App Router defaults to Server Components. Use `"use client"` at the top of the file *only* when you need interactivity (hooks, state, event listeners).
- **Data Fetching**: Avoid direct `fetch` or `axios` calls inside components. **Always** use React Query via custom hooks. This ensures caching, deduplication, and optimistic updates are handled globally.
- **Styling**: Stick to Tailwind utility classes. For complex conditional logic, use the `cn()` utility (`lib/utils.ts`) which merges classes properly with `clsx` and `tailwind-merge`.
- **Forms**: Use `react-hook-form` paired with `zod` for validation.
- **Icons**: Use `lucide-react` for all iconography to maintain visual consistency.
